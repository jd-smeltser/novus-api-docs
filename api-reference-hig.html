Authentication

The Novus API uses custom header authentication with temporary tokens for secure access.

⏱️
Token Expiration: Authentication tokens are valid for exactly 5 minutes. Build your workflows with this constraint in mind.

Authentication Flow

  1. Request Token: Send credentials via headers to /Token endpoint
  2. Extract Token: Parse the token from the JSON response
  3. Use Token: Include token in headers for API requests
  4. Monitor Expiration: Refresh token before 5-minute limit

Header Authentication

Headers Required for All Requests

Token Request Headers

Header Type Required Description
UserName string Required Your assigned Novus API username
Password string Required Your assigned Novus API password

All API Request Headers

Header Type Required Description
UserName string Required Your assigned Novus API username
Password string Required Your assigned Novus API password
Token string Required Valid authentication token from /Token endpoint

Base URLs

The API is available in both QA and Production environments with specific URL patterns.

QA Environment

Testing

Base URL: https://[CLIENT_URL]/[API_PATH]/WAEPANYL

Token Endpoint: /WAEPANYL/Token

Member Info: /WAEPANYL/GetMemberInfo

Use for development, testing, and integration validation.

Production Environment

Live

Base URL: https://[CLIENT_URL]/[API_PATH]

Token Endpoint: /WAEPANYL/Token

Member Info: /GetMemberInfo

⚠️
Key Difference: Production GetMemberInfo omits /WAEPANYL/ from the path.

Security & Limits

🔒
IP Restriction: All API access is restricted by IP address. Your IP must be whitelisted before you can make any requests.

Access Requirements

  • IP Whitelisting: Required for all environments and endpoints
  • Token Expiration: 5 minutes from generation time
  • HTTPS Only: All requests must use secure HTTPS protocol
  • Header Authentication: Custom headers required for all requests

Rate Limits

Contact your API administrator for specific rate limiting information for your implementation.

Data Models

Detailed specifications for all response data structures.

Demographics Object

Complete member demographic information when available.

Field Type Description Example
firstName string Member's first name "Jane"
lastName string Member's last name "Doe"
dateOfBirth string ISO 8601 datetime with timezone "1999-03-19T00:00:00"

emailAddressInsured string|null Member's email address if available "jane.doe@company.com"
membershipStatus string Current membership status "ACTIVE"
maritalStatus string Member's marital status "Single"
residentialAddress object Address object (see below) Address Object

Address Object

Residential address information structure.

Field Type Description Example

Field Type Description Example

policyNumber string Internal policy number "GTL-130601-01"
productCode string Product type identifier "GTL"

Error Handling

Complete reference for HTTP status codes and error responses.

HTTP Status Codes

Status Code Meaning Common Cause Recommended Action
200 OK Request successful Process response data normally
400 Bad Request Invalid JSON or missing required fields Validate request format and required parameters
401 Unauthorized Invalid credentials or expired token Refresh authentication token or verify credentials
403 Forbidden IP address not whitelisted Contact administrator for IP approval
404 Not Found Incorrect endpoint URL Verify endpoint path and environment
429 Too Many Requests Rate limit exceeded Implement request throttling and retry logic
500 Internal Server Error Server-side processing error Retry after delay; contact support if persists

Error Response Format

When errors occur, the API returns structured error information to help with debugging.

Error Response Example 
{
  "error": {
    "code": "INVALID_TOKEN",
    "message": "The provided authentication token has expired or is invalid",
    "details": "Token expired at 2024-03-15T10:30:00Z",
    "timestamp": "2024-03-15T10:35:00Z"
  }
}

Error Object Fields

Field Type Description
code string Machine-readable error code
message string Human-readable error description
details string Additional context about the error
timestamp string ISO 8601 timestamp when error occurred
